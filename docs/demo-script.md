# 验收演示脚本

## 场景设定
- 订单号：`DD202604290001`
- 商品：`无线蓝牙耳机`
- 状态：`已签收，暂无售后`
- 下单日期：`2026-04-29`
- 金额：以演示库订单金额为准

## 五步演示流程
1. 打开“咨询工作台”，绑定订单 `DD202604290001`，展示当前模型、订单上下文和知识库状态。
2. 输入“这个订单能不能退货？”，强调页面会返回意图标签、订单结合判断和规则依据。
3. 继续追问“那退款一般多久到账？”，展示系统识别这是追问，并从退货语境切换到退款进度意图。
4. 再问“如果商家一直不处理怎么办？”，展示系统从多轮上下文升级到投诉/人工转接意图。
5. 打开“人工工单”页面，展示自动生成的工单号、优先级、AI 摘要、处理建议和状态流转。

## 答辩讲解要点
- 这不是普通聊天框，链路是“多轮对话 -> 意图识别 -> 知识库检索 -> 结合订单上下文生成答案”。
- 所有回答都能在右侧看到上下文承接、规则依据、AI 生成状态和工单判定过程，证明做了可解释的检索融合。
- 投诉和异常问题会自动生成工单，也可以由客服手动点击“转人工”，体现真实客服系统闭环。
- 即使没有配置 OpenAI Key，也能使用本地规则模板稳定演示，避免现场网络问题影响验收。
