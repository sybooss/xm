## 五步演示流程
1. 使用 `admin / 123456` 登录系统，说明后台操作具备管理员权限控制。
2. 打开“咨询工作台”，绑定订单 `DD202604290001`，展示当前模型、订单上下文和知识库状态。
3. 输入“这个订单能不能退货？”，强调消息立即发出，页面会流式展示处理进度并返回意图标签、订单判断和规则依据。
4. 继续追问“那退款一般多久到账？”，展示系统识别这是追问，并从退货语境切换到退款进度意图。
5. 再问“如果商家一直不处理怎么办？”，展示投诉/人工转接意图、自动工单，以及“人工工单”页面里的优先级、AI 摘要、处理建议和状态流转。

## 答辩讲解要点
- 这不是普通聊天框，链路是“多轮对话 -> 意图识别 -> 知识库检索 -> 结合订单上下文生成答案”。
- 所有回答都能在右侧看到上下文承接、规则依据、AI 生成状态和工单判定过程，证明做了可解释的检索融合。
- 投诉和异常问题会自动生成工单，也可以由客服手动点击“转人工”，体现真实客服系统闭环。
- 即使没有配置 OpenAI Key，也能使用本地规则模板稳定演示，避免现场网络问题影响验收。
