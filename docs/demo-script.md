# 验收演示脚本

## 场景设定
- 订单号：`YT20260415001`
- 商品：`AirTune 蓝牙耳机`
- 状态：`待商家审核`
- 下单日期：`2026-04-15`
- 金额：`299 元`

## 五步演示流程
1. 打开“客服对话台”，展示系统概览、当前引擎模式和知识库数量。
2. 输入“我刚买的蓝牙耳机不想要了，可以退货吗？”，强调页面会返回意图标签、订单结合判断和规则依据。
3. 继续追问“那退款一般多久到账？”，说明系统保留了多轮上下文，同时切换到退款进度意图。
4. 再问“我的包裹 48 小时没更新物流了怎么办？”，展示系统能切换到物流查询并命中异常催件规则。
5. 最后输入“我想转人工投诉商家一直不处理”，展示投诉/人工转接意图以及升级处理依据。

## 答辩讲解要点
- 这不是普通聊天框，链路是“多轮对话 -> 意图识别 -> 知识库检索 -> 结合订单上下文生成答案”。
- 所有回答都能在右侧看到规则依据，证明做了检索融合。
- 即使没有配置 OpenAI Key，也能使用本地规则模板稳定演示，避免现场网络问题影响验收。
