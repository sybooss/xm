# 验收演示脚本

## 场景设定
- 订单号：`DD202604290001`
- 商品：`无线蓝牙耳机`
- 状态：`已签收，暂无售后`
- 下单日期：`2026-04-29`
- 金额：以演示库订单金额为准

## 七步演示流程
1. 先用 `admin / 123456` 登录进入“答辩展示中心”，用该页面总览系统主题、演示顺序、8 个已实现亮点和现场检查清单。
2. 在登录页展示“注册”入口：注册一个客户账号后进入咨询工作台，说明普通客户只能访问客户侧能力；再退出并使用管理员账号登录，说明后台操作具备管理员权限控制。
3. 打开“咨询工作台”，绑定订单 `DD202604290001`，展示当前模型、订单上下文和知识库状态。
4. 输入“这个订单能不能退货？”，强调消息立即发出，页面会流式展示处理进度并返回意图标签、订单判断和规则依据。
5. 继续追问“那退款一般多久到账？”，展示系统识别这是追问，并从退货语境切换到退款进度意图。
6. 再问“如果商家一直不处理怎么办？”，展示投诉/人工转接意图、自动工单，以及“人工工单”页面里的优先级、AI 摘要、处理建议和状态流转。
7. 最后进入“日志诊断中心”，用 AI 成功率、平均耗时、知识命中、高频命中文档和处理轨迹步骤收尾，说明系统不是黑盒聊天，而是有可解释证据链。

## 答辩讲解要点
- 这不是普通聊天框，链路是“多轮对话 -> 意图识别 -> 知识库检索 -> 结合订单上下文生成答案”。
- “答辩展示中心”不是静态宣传页，它集中链接到咨询工作台、知识库、工单、日志和 AI 测试，方便老师按一条主线检查真实功能。
- 所有回答都能在右侧看到上下文承接、规则依据、AI 生成状态和工单判定过程；日志诊断中心还能汇总 AI 成功率、平均耗时、知识命中和轨迹步骤，证明系统可解释、可追踪。
- 投诉和异常问题会自动生成工单，也可以由客服手动点击“转人工”，体现真实客服系统闭环。
- 客户注册后默认是 `CUSTOMER`，不能进入系统总览、知识库、订单管理、人工工单、日志中心和 AI 测试等后台页面。
- 即使没有配置 OpenAI Key，也能使用本地规则模板稳定演示，避免现场网络问题影响验收。
