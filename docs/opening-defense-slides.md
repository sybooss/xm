# 开题答辩 PPT 文案（7页）

题目：基于检索增强的电商退换货智能客服系统设计与实现  
建议总时长：6 分钟左右

## 第1页 课题背景与研究意义

**标题**  
基于检索增强的电商退换货智能客服系统设计与实现

**页面要点**
- 电商平台售后咨询量大，退货、换货、退款、物流异常问题高频出现。
- 人工客服成本高，处理标准不一致，用户等待时间长。
- 通用聊天机器人在强规则场景下容易回答不稳、缺少依据。
- 因此需要构建面向退换货场景的领域智能客服系统。

**建议配图**
- 左侧放电商售后流程图
- 右侧放“人工客服痛点”关键词：高并发、规则复杂、响应慢、可解释性弱

**你要说的话**
- 先讲场景痛点，不要一上来讲模型。
- 强调本课题属于“复杂软件系统中的领域客服应用”。

**建议时长**
- 40 秒

**标题**  
国内外研究现状与存在问题

**页面要点**

## 第4页 系统总体架构与技术路线

**标题**  
系统总体架构与技术路线

**页面要点**
- 前端：React + Vite
- 后端：FastAPI
- 数据层：SQLite 本地知识库 + 会话记录
- 核心链路：用户问题 -> 意图识别 -> 知识库检索 -> 订单上下文融合 -> 回复生成 -> 返回依据与轨迹
- 生成策略：OpenAI / 本地规则双模式

**建议图示**

```text
用户输入
   ↓
React 前端交互页
   ↓
FastAPI 接口层
   ↓
意图识别模块（7类）
   ↓
知识库检索模块（SQLite）
   ↓
订单上下文融合
   ↓
回复生成（OpenAI / Local）
   ↓
回答 + 规则依据 + 执行轨迹
```

**你要说的话**
- 明确强调这不是单一聊天框，而是一个多模块协同的软件系统。

**建议时长**
- 60 秒
